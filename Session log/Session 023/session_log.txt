A) BLOCK 10 — CONNECTORS  (commit 6be73a7)

   5 slices:
   1. SVG overlay drawing connector lines from node centers with arrow
      markers (<marker id="journey-arrow">). Invisible 14px transparent
      hit-lines sit behind the visible 2px lines for click-to-delete
      (same pattern as CanvasEditPage:1019).

   2. Connector label layer — HTML (not SVG) buttons positioned absolutely
      at the midpoint of each connector, so label interaction can use real
      inputs/buttons. Empty labels show a dashed placeholder pill that
      reads "label"; set labels show a solid white pill.

   3. Connect mode — new "Connect" button in the toolbar. Click a source
      node → click a target node → connector is created. Rules enforced:
        - Start nodes cannot be targets
        - End nodes cannot be sources
        - Directed dedup (only A→B, not undirected like Canvas)
        - Clicking the source again cancels
        - Esc cancels
      While in connect mode the workspace cursor is crosshair and the
      source node gets a colored ring using COLORS[type].ring. Connector
      label layer is hidden while in connect mode so clicks reach nodes.

   4. Inline label edit — click a label pill to open an input at that
      position (autofocus, Enter saves, Esc cancels, blur saves). A small
      × button next to each label deletes the connector outright.

   5. Build + test + commit. Blocks 10-12 re-use the pre-existing
      beforeunload warning and dirty badge from Block 7.

B) BLOCK 11 — UNDO HISTORY  (commit 4683be3)

   Ported the past-stack pattern from CanvasEditPage one-for-one.

   - New HistorySnapshot type = { nodes, connectors }
   - past: HistorySnapshot[] state, capped at HISTORY_LIMIT (50)
   - pushHistory() helper, called BEFORE every coarse mutation:
       addOfferingNode, addDecisionNode, addStatusNode, addEndNode,
       handleConnectClick (on successful add), deleteConnector,
       saveConnectorLabel (only when label actually changed).
   - Drag uses the lazy-push pattern: dragStateRef captures prevNodes
     and prevConnectors on pointerdown, and the snapshot is only pushed
     on the FIRST real pointermove. A pointerdown with no movement does
     not consume an undo slot.
   - undo() pops the top snapshot, restores nodes+connectors, clears any
     in-progress label edit and connect mode, marks dirty.
   - Cmd/Ctrl+Z keyboard shortcut wired via the existing keydown useEffect
     (the one that already handled Escape). Typing in inputs/textareas is
     ignored. past stack is reset to [] on fresh flow load.
   - Undo button in the toolbar, disabled when past is empty.

C) BLOCK 12 — SAVE FLOW  (commit 6632cbf)

   - saveFlow(): PATCHes journey_flows.content with { nodes, connectors },
     clears dirty, updates local flow state (optimistic updated_at), emits
     a journey_flow 'updated' audit log entry with details sub='content',
     node_count, connector_count. Toasts success/error.
   - Primary Save button in the header, disabled when !dirty or saving,
     shows "Saving…" during the request.
   - Cmd/Ctrl+S keyboard shortcut (always preventDefault, only fires when
     dirty and not already saving).
   - dirty badge and beforeunload warning were already wired from Block 7.
