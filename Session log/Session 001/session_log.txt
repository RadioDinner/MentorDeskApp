DIRECTIONAL DECISIONS
----------------------

1. STAFF TABLE DESIGN: staff.id is a standalone UUID, not the same as
   auth.users.id. user_id is a nullable FK that gets populated separately
   when login access is granted. This allows staff to exist in the system
   without a login account.

2. ROLE SCOPE: Three staff roles in scope for now: admin, mentor, staff.
   Mentee is a separate concept, handled separately later (Phase 2+).

3. TAILWIND v4: Using the new @tailwindcss/vite plugin (not the legacy
   tailwind.config.js approach). CSS entry is just `@import "tailwindcss"`.

4. VERCEL CONFIG: Managed via vercel.json at repo root rather than Vercel
   dashboard settings, so it's version-controlled.

5. SUPABASE INIT SAFETY: The Supabase client uses a placeholder URL/key
   fallback so the app renders even before env vars are set. Once the
   Supabase project is connected, real values go in Vercel env vars.
