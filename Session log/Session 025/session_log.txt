DELIBERATE DESIGN DECISIONS
----------------------------

  - Status and End node types kept in the type system and render layer
    for backwards compatibility with existing flows. They just can't be
    created from the toolbar anymore.
  - isEnd is optional (undefined = false) so all existing node data
    works without migration.
  - The auto-layout algorithm uses longest-path depth (not shortest)
    so nodes reachable via multiple paths always appear after all
    predecessors.
  - Meeting cancellation uses Option B (always allow, warn about credit
    loss) rather than Option A (block late cancels).
