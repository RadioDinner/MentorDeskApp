DIRECTIONAL DECISIONS
----------------------
- First shared primitives live under `app/src/components/ui/`.
  Future primitives (Skeleton, Toast, Drawer, PageBar, Input,
  Select, TextArea) should go in the same directory and be
  re-exported from `components/ui/index.ts`.
- Formatters live in `app/src/lib/format.ts` rather than
  `components/ui/` because they're pure functions, not JSX.
- The Modal primitive is intentionally simple — not a full focus
  trap, but role=dialog + aria-modal + Escape + focus restore +
  body scroll lock. A real focus trap is a follow-up if we find
  it matters.
- Not migrating all 30+ pages in one commit — too much review
  surface, too much regression risk. Plan is incremental migration
  as surrounding work touches each page.
- Kept the tiny `ActionButton` sub-component inside InvoicingPage
  (5 lines) because its sizing (px-2.5 py-1 text-[11px]) is
  smaller than the standard Button.sm. Didn't want to contort
  the primitive to fit.
- The InvoicePrintPage status pills (uppercase, border, print-
  specific) were left as inline styling. They're print design,
  not a repeating list-UI pattern, so Badge migration would be
  a lossy fit.
