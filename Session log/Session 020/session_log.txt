This session can be split into four blocks, in the order they were

BLOCK 1 — CANVASES v1.1 (the deferred CanvasEditPage rewrite)

   Per the Session 019 handoff, all Tier-1 Canvas v1.1 features were
   scaffolded (types + lib/canvas.ts + CanvasCreatePage seed) but the
   big CanvasEditPage rewrite was deferred. This session finished it
   in slices:

   1. Phase A — consolidate (commit 538f891)
      CanvasEditPage now imports COLORS, colorClasses, clientId,
      NOTE_DEFAULTS, NOTE_MIN, WORKSPACE_SIZE, HISTORY_LIMIT, and the
      create*/migrate* helpers from lib/canvas.ts. The duplicate
      declarations at the top of the file (called out as a cleanup
      task in the Session 019 handoff) are gone.

   2. Undo history (commit 33c6d6f)
      Past stack capped at HISTORY_LIMIT, snapshot taken BEFORE
      coarse actions (add, delete, drag start, color change, edit
      start, etc.). Cmd/Ctrl+Z restores. No redo (deliberately
      tabled, per the design notes).

   3. Resizable notes (commit 46807a9)
      Bottom-right resize handle on every note. Separate
      resizeStateRef from dragStateRef, both share the same
      document-level pointermove/pointerup listeners. Clamped to
      NOTE_MIN (140 x 100) and WORKSPACE_SIZE bounds.

   4. Multi-select + bulk move + keyboard shortcuts (commit d562e28)
      Shift-click adds/removes from selectedIds. Plain click =
      clear-then-select. Click empty canvas = clear. When dragging a
      selected note, all selected notes move together using a
      startPositions map keyed by noteId. Selected notes get a ring.
      Global keydown listener (scoped out of input/textarea targets)
      handles N (new sticky), Delete/Backspace (delete selected),
      Cmd/Ctrl+Z (undo).

   5. Phase B — checklist, link, connectors, rich text (commit 882ac85)
      The big one. CanvasEditPage now handles all three note types
      from the discriminated union plus the connector layer:

        · Sticky notes render via dangerouslySetInnerHTML using
          renderMarkdown(text) from lib/canvas. Edit mode is a plain
          textarea on the markdown source. The optional B/I/• toolbar
          was skipped for V1 — raw markdown typing works fine and
          the toolbar is the finicky part.

        · Checklist notes: title input + rows of [checkbox] [text]
          [×]. Enter in a row creates a new row below it. "+ Add item"
          link at the bottom. Toggling a checkbox pushes history.

        · Link notes: type dropdown (course / habit / canvas / url)
          with a value picker. Lists for course/habit/canvas are
          lazy-loaded the first time the picker opens and cached in
          state. Display is a small icon-card; click opens the
          target (/courses/:id/edit, /habits/:id/edit, /canvases/:id,
          or external URL in a new tab).

        · Connectors: toolbar "Connect" button enters connectorMode.
          First click sets sourceId, second click on a different note
          creates the connector and exits the mode. Escape or empty
          click cancels. Render is an SVG overlay over the workspace,
          line drawn from note-center to note-center, recomputed from
          the current note state every render so moving a note drags
          its connectors. Click a line to delete.

      CanvasEditPage was NOT extracted into per-type child components
      (StickyNoteCard, ChecklistNoteCard, etc.) as suggested in the
      Session 019 design notes. The single-file approach is around
      ~1100 lines but readable; extracting can happen later if it
      causes pain.
