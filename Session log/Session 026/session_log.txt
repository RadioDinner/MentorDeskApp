DELIBERATE DESIGN DECISIONS
----------------------------

  - advanceTrigger / delayValue / delayUnit are stored per-node in the
    journey template JSON (not in a separate DB table). This means the
    settings travel with the flow and are snapshot-copied into mentee
    journeys. The actual automation that reads these settings and
    triggers advances is not yet built — the settings are the data
    model prep for it.

  - The node settings sidebar is inside JourneyEditPage (not a separate
    component file) because it's tightly coupled to the editor state
    (nodes, connectors, updateNode).

  - Mentor Tasks use the pairings table to determine which mentor(s) to
    notify when a journey hits a decision node. If a mentee has multiple
    mentors (via multiple pairings), all get a task.

  - cancelled_by FK was dropped entirely rather than changed to a
    polymorphic reference. The column is a UUID that can hold either a
    staff or mentee ID — the app layer knows which based on context.

  - Module key rename from 'flows' to 'journeys' is handled in the
    frontend only (Sidebar + ModuleAccessControl normalize). No DB
    migration to update existing allowed_modules arrays — the frontend
    mapping handles it transparently.
