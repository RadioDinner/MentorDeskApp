7. NAVIGATION BUG FIX — STALE LOADING STATE:
   - Root cause: useEffect dependencies used full `profile` object

     with identical data, causing effects to re-fire and race with
     in-flight fetches, leaving pages stuck on "Loading..."
   - Fix: Changed all 12 affected pages from [profile] to
     [profile?.organization_id] (stable string primitive)
   - Files fixed: DashboardPage, CoursesPage, EngagementsPage,
     MenteesListPage, PairingCreatePage, CompanySettingsPage,
     ProfilePage, PairingsPage, AuditLogPage, PeopleListPage,
     OfferingsPage

DIRECTIONAL DECISIONS
----------------------

1. MODULE RESTRUCTURE: Offerings split into Courses and Engagements as
   separate modules under a new "Offerings" group. This gives each its
   own sidebar entry, independent permission control, and room to grow
   (Course Builder is course-specific, not engagement-relevant).

2. PERMISSION GROUPS AS PRESETS: Permission groups are organizational
   templates that quick-apply module sets, NOT enforced role definitions.
   Individual staff can still have custom module access that differs from
   any group. Groups are a convenience, not a constraint.

3. PAIRINGS GRID AS DEFAULT: The spreadsheet grid view is now the default
   tab on the Pairings page, with the existing By Mentor, Unpaired, and
   By Status views preserved as alternate tabs.

4. COURSE BUILDER ARCHITECTURE: Lessons are stored in a dedicated
   `lessons` table linked to offerings via offering_id (where type='course').
   Questions are in `lesson_questions` with question_type discriminator
   ('quiz' with options/correct answers, or 'response' for open-ended).
   Per-lesson due dates are an org-level opt-in feature.

5. AUTH EFFECT STABILIZATION: All page-level data fetching effects now
   depend on profile?.organization_id instead of the full profile object,
   preventing spurious re-fetches on Supabase token refresh events.
