cleaned out completely (it previously contained an entire app codebase from

DIRECTIONAL DECISIONS
----------------------
1. FRESH START: The repo was completely wiped. No code from the prior
   CourseCorrect project is being carried forward.

2. PRODUCT MODEL: MentorDesk is a SaaS tool licensed to coaching
   organizations -- similar to how Microsoft licenses M365. We are NOT
   building org provisioning or a licensing portal at this time.

3. SINGLE ORG FOCUS: Building for a single org in function right now.
   Test org is called "TechBeans." But the data model will include
   organization_id on all relevant tables from day one.

4. BRANDING: Shopify-style. Orgs can customize logo and color theme.
   MentorDesk remains the visible product name and brand.

5. SAME APP FOR ALL ROLES: The mentee experience lives in the same app
   as the admin/mentor/staff experience. Role-based routing determines
   what each user sees.

6. TECH STACK: React + Vite frontend, Supabase backend (PostgreSQL,
   auth, storage), Vercel for deployment.

7. DEPLOYMENT: Vercel auto-deploy from main branch. Preview deployments
   per branch/PR planned for later.

8. FUTURE BACKLOG ITEM: Per-org release versioning (ability to put
   different orgs on different app versions). Not in scope now.
