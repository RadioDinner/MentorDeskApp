Date: 2026-04-06


SUMMARY
--------
This was the inaugural session for the MentorDesk project. The repository
previously contained an entire codebase from a prior project called
"CourseCorrect" that had been pushed here by mistake. We wiped it completely
and started fresh. No application code was written -- this session was
entirely about cleaning the repo, establishing project conventions, defining
terminology, and building the roadmap.


CHANGES MADE
-------------
1. Wiped the repo clean -- removed all old CourseCorrect/MentorDesk code,
   migrations, android app, super-admin app, landing page, supabase
   functions, and all other files from the prior project.

2. Created `new_session_instructions` -- persistent file at repo root
   defining how every session should be managed:
   - Create a numbered session folder in "Session log/"
   - Log every user prompt to prompt_history.txt
   - Write a handoff document (session_log.txt) at end of session
   - Read the previous session's handoff at start of next session

   - Product terms: App, Web App, iOS App, Android App, MentorDesk
   - User roles: Organization, Admin, Mentor/Coach, Mentee, Staff

   - Branding: Org Branding, Theme

6. Deleted stale remote branches, leaving only main.


DIRECTIONAL DECISIONS
----------------------
1. FRESH START: The repo was completely wiped. No code from the prior
   CourseCorrect project is being carried forward. We are building
   everything from scratch.

2. PRODUCT MODEL: MentorDesk is a SaaS tool that will be licensed to
   coaching organizations -- similar to how Microsoft licenses M365.
   We are NOT building org onboarding, provisioning, or a super-admin
   dashboard at this time.

3. SINGLE ORG FOCUS: Building for a single org in function right now.
   Test org is called "TechBeans." But the data model will include
   organization_id on all relevant tables from day one so we don't
   have to retrofit multi-tenancy later.

4. BRANDING MODEL: Shopify-style. Organizations can customize their
   logo and color theme within the app. MentorDesk remains the visible
   product name and brand -- this is NOT a full white-label where the
   platform name is hidden.

5. SAME APP FOR ALL ROLES: The mentee experience lives in the same
   app/codebase as the admin/mentor/staff experience. Role-based
   routing determines what each user sees. We are not building a
   separate mentee-facing app.

6. TECH STACK DECIDED:
   - Frontend: React + Vite
   - Backend: Supabase (PostgreSQL, auth, storage, edge functions)
   - Deployment: Vercel, auto-deploy from main branch
   - Styling framework: TBD (Tailwind suggested but not confirmed)
   - TypeScript vs JavaScript: TBD

7. DEPLOYMENT: Vercel auto-deploy from main branch for now. Preview
   deployments per branch/PR planned for later.

8. FUTURE BACKLOG ITEMS NOTED:
   - Per-org release versioning (ability to put different orgs on
     different releases of the app)
   - Preview deployments on Vercel
   - Super-admin dashboard
   - Org onboarding/provisioning flow
