DECISIONS / ARCHITECTURE NOTES
--------------------------------

- Toast is the right pattern for mutation feedback (save/update/
  send-email/etc). Inline banners remain appropriate only for
  persistent page-level errors where the user needs to take action
  (e.g. a fetch error with a Retry button — those use LoadingErrorState
  which is still in place).

- Skeleton primitive exists but is not yet adopted anywhere. Next
  session can wire it into list-page "Loading..." states.

- EngagementManageModal and MenteeManageSlideOver still have their
  own raw overlay divs (not converted to the Modal primitive). They
  were migrated for button and format primitives, but the overlay
  conversion is a larger refactor noted in the backlog under item #7
  (Accessibility pass).
