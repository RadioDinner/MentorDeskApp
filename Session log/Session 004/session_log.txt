DIRECTIONAL DECISIONS
----------------------

1. ARCHIVE BEFORE DELETE: People must be archived before they can be
   deleted. No direct delete from active state. Two-step confirmation
   on delete.

2. MULTI-ROLE VIA MULTIPLE RECORDS: A single user_id can have multiple
   staff records (one per role) and a mentee record. This is a data-level
   approach, not a permissions flag — each role is a full record with
   its own settings.

3. TOKEN REFRESH ISOLATION: AuthContext no longer re-fetches profiles on
   Supabase token refresh events. Only fetches on SIGNED_IN and
   INITIAL_SESSION. This prevents the Loading... bug.

4. AUTO-DELETE IS CLIENT-SIDE: Archive auto-purge runs on login from
   the browser. No server-side cron. This means purging only happens
   when someone logs in.

5. MODULE ICONS: Gradient letter circles are the icon system. Each
   module has a unique letter and two-tone gradient color.
